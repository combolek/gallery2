Gallery Remote roadmap

Document goal

Serve as a basis for team and community discussions about the future of Gallery Remote and loosely synchronize its development to that of Gallery 2.

2.0 [Summer 2003, in synch with G2 release]

The goal of this release is to open up Gallery to all kinds of remote apps, via the creation of SOAP bindings into the core.

Most or all Core functionality would be exposed as SOAP calls. A Java Gallery SDK would be created, which would replicate PHP objects to allow remote Java applications to perform actions on a gallery remotely and without having to worry about SOAP. Gallery Remote 2 (GR2) would be one such app.

Usage case scenarios for GR2 include:

  • reordering existing albums
  • moving pictures between galleries off-line
  • creating off-line galleries
  • editing comments and images

One of the design constraints for GR2 is to not delay G2 by not requiring Core modifications. I hope that's possible.

2.5 [Winter 2003]

In this release, we include the possibility to influence Core development to support some of the features that were not supported by Gallery 2.0's core.

All contributors' submissions can be hosted on the Community gallery and winner will be credited in Readme and about box.
